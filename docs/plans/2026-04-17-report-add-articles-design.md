# 日报手动加入文章设计文档

**日期：** 2026-04-17

## 1. 目标

在现有“日报编辑后再发布”能力上，再补一层人工补稿能力，让后台可以把未入选、但已经处理完成的文章手动加入当前日报。

本轮目标：

- 在日报编辑器里看到候选文章
- 支持把候选文章加入日报
- 保存后进入当前日报条目列表
- 重新发布后公开页能显示新增文章

本轮不做：

- 跨日期文章搜索
- 从全量文章池高级筛选
- 拖拽式编排
- 批量加入

## 2. 方案对比

### 方案 A：日报详情接口直接返回候选文章

后端在日报详情里直接附带：

- 当前日报条目
- 同一天、已处理成功、未入选的候选文章

优点：

- 前端实现最简单
- 不用新增独立搜索接口
- 数据天然和当前日报强相关

缺点：

- 候选范围只适合同日报日期

### 方案 B：新增文章搜索接口

在日报编辑器里独立搜索全部文章，再手动加入。

优点：

- 更灵活

缺点：

- 实现明显更重
- 本轮容易把范围拉大

## 3. 采用方案

本轮采用 **方案 A**。

## 4. 候选文章规则

候选文章必须满足：

- `article.content.ai_status == "success"`
- `article.status == "processed"` 或可用于日报
- 文章日期与 `report.report_date` 一致
- 当前不在该日报中

## 5. 后端设计

### 5.1 日报详情补候选列表

在日报详情接口里增加 `candidate_articles` 字段。

### 5.2 保存接口支持新条目

现有 `PUT /admin/api/reports/{id}` 的 `items[]` 扩展为：

- 已存在条目：带 `id` 和 `article_id`
- 新增条目：`id = null`，只传 `article_id`

后端保存时：

- 已存在条目更新顺序和分组
- 新条目新建 `DailyReportItem`
- 未出现在 payload 中的旧条目删除

### 5.3 约束

- 同一日报里同一篇文章不能重复
- 非可发布文章不能加入日报

## 6. 前端设计

在现有日报编辑器里增加一块“候选文章”区域：

- 一个搜索框，只做前端过滤
- 候选卡片列表
- 点击“加入日报”即可加入编辑列表

候选列表只在当前编辑日报时出现。

## 7. 风险与处理

### 7.1 重复加入同一篇文章

前端和后端双重去重。

### 7.2 新增文章后候选列表不同步

保存成功后以后端返回的新详情为准，重新刷新候选列表。

### 7.3 候选文章太多

本轮后端可限制候选数，例如 30 篇，足够本地使用。

## 8. 验证

需要覆盖：

- 日报详情返回候选文章
- 保存时可以加入新文章
- 重复文章会被拒绝或忽略
- 重新发布后公开页出现新增文章
